## 3. Orchestration Rules (actual code)

### _decide_next_action (status API)

| State | Condition | next_action | can_auto_continue |
|-------|-----------|-------------|-------------------|
| draft | — | generate_ticket | true |
| ticket_ready | — | dispatch | true |
| dispatched | no AgentRun | create_agent_run | true |
| dispatched | AgentRun queued/running | wait_agent_result | false |
| dispatched | AgentRun succeeded | submit_result | true |
| dispatched | AgentRun failed/canceled | agent_failed | false |
| result_submitted | — | start_review | true |
| reviewing | has auto-approvable decision | (auto-approve in step) | true |
| reviewing | no decision, evaluate → human_required | evaluate_approval | false |
| human_required | — | — | false |
| approved | — | — | false |
| archived | — | — | false |

### orchestration_step

| State | Action | Stop Reason |
|-------|--------|-------------|
| draft | generate_ticket → ticket_ready | — |
| ticket_ready | dispatch → dispatched | — |
| dispatched + no run | create+start AgentRun | waiting_for_agent_result |
| dispatched + succeeded run | submit_result + create artifacts | — |
| dispatched + failed/canceled run | — | agent_failed |
| result_submitted | start_review → reviewing | — |
| reviewing + auto-approvable decision | auto_approve → approved | — |
| reviewing + no decision | evaluate → human_required | human_required |
| human_required | — | human_required |
| approved | — | approved_waiting_archive |

### reviewing auto-approve

1. First queries existing `ApprovalDecision` with `task_id == task.id`, `auto_approve_allowed == True`, `human_required == False`.
2. If found and `risk_level == 'low'`, calls `auto_approve` directly.
3. Otherwise falls back to `evaluate_approval`.
4. Cross-task decision isolation: query always includes `task_id == task.id`.
