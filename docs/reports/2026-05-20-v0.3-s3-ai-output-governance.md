## 1. 完成范围

### S3-1: AI 输出校验
- `validate_agent_run_result()` 综合校验：空输出、输出大小、JSON 合法性
- malformed raw_result_json → valid=False, requires_human=True
- governance flag（secret pattern / forbidden path）→ warnings, not errors（不硬失败，走 requires_human）

### S3-2: patch.diff 校验（fnmatch）
- `validate_patch_diff()` 使用 fnmatch 匹配禁止路径
- 从 `diff --git` / `---` / `+++` 行提取路径
- 校验：diff --git 头、空内容、secret 模式（7 种 pattern）、禁止路径（.env, .pem, .key, migrations, CI config, credentials 等）

### S3-3: review.md → AgentReview
- `parse_review_result()` 提取 decision / risk_level / confidence
- `create_agent_review_from_ai_output()` 创建 AgentReview 记录（不直接 approve）
- high risk / changes_requested / malformed 不得自动 approve

### S3-5: risk_report → RequiresHuman 链路
- `check_risk_report()` 校验 risk_level
- high/critical → requires_human=True
- malformed risk_report → requires_human=True + errors
- 已接入 dispatch_agent_run，通过 ApprovalDecision 记录

### S3-6: AgentRun trace
- `build_trace_json()` 标准化 raw_result_json → provider_raw + governance + trace

### Secret redaction
- `redact_secrets()` 模块级函数，覆盖 7 种 pattern
- dispatch_agent_run 中自动对 provider_raw 做脱敏

## 2. 真实流程接入口

| 路径 | 接入方式 |
|------|---------|
| dispatch_agent_run | succeeded 后调用 validate_agent_run_result → 写 trace → 创建 AgentReview → 创建 artifact |
| requires_human | 自动创建 ApprovalDecision(human_required=True, auto_approve_allowed=False) |
| secret redaction | provider_raw 写入前自动脱敏 |
| review_md | 自动 parse → create_agent_review_from_ai_output |

### Artifact secret redaction

- `_create_artifacts_from_result()` 在创建 artifact 前对全部字段调用 `redact_secrets()`
- 覆盖：plan_md / patch_diff / review_md / raw_result_json
- sha256 和 size_bytes 基于 redacted content 计算
- raw_result_json artifact 使用 `run.raw_result_json`（已 redacted），不使用原始 `result.raw_result_json`
- 集成测试 `test_artifact_secret_redaction` 验证 artifact 不含 fake secret，含 `***REDACTED***`
