# v0.4 S1 — AI Coding Sandbox

- **Date**: 2026-05-20
- **Branch**: `feature/v0.4-s1-ai-coding-sandbox`
- **Status**: Complete

## Scope

Implemented the "AI writes code" pipeline in secure sandbox mode. This phase adds Code Context, AI patch generation with file awareness, and Patch Apply Sandbox — all in-memory, no real file writes, no git/CI/PR.

### New Capabilities

1. **Code Context Service** (`code_context_service.py`)
   - Store/retrieve file context as TaskArtifacts (`code_context_bundle` type)
   - API: `POST /api/tasks/{id}/code-context`, `GET /api/tasks/{id}/code-context`
   - No Project.root_path, no local file system, no shell execution

2. **AI Provider Code Context Integration**
   - `SandboxProvider._execute_result()` generates contextualized patch.diff referencing actual code context files
   - `OpenAIProvider._build_prompts()` injects file contents into EXECUTE prompts
   - Both providers respect `code_context=None` fallback

3. **Patch Apply Sandbox** (`patch_apply_sandbox_service.py`)
   - Parses unified diff (`diff --git` format)
   - Applies patches in-memory on virtual file system from code context
   - Generates `PatchApplyReport` with changed files, sha256, before/after previews
   - Creates artifacts: `patch_apply_report`, `changed_files_summary`, `changed_file_preview`
   - API: `POST /api/tasks/{task_id}/agent-runs/{run_id}/sandbox/apply-patch`
   - API: `GET /api/tasks/{task_id}/sandbox/patch-results`

4. **Governance Integration**
   - Validates patch via existing `validate_agent_run_result()` before applying
   - Forbidden paths, secrets, oversized patches all blocked
   - Secret redaction via `redact_secrets()` on artifact content
   - Archived tasks forbidden from code context and sandbox operations

## Security Boundaries

- No shell/subprocess/os.system calls
- No Project.root_path or local file system access
- No git clone/commit/push/apply
- No GitHub PR creation
- No CI/Sonar/Deploy API calls
- No secret_ref access
- Secrets redacted via `redact_secrets()` before artifact storage
- All patch application is in-memory only
