## 核心能力

### 后端

| 模块 | 能力 |
|------|------|
| **多项目管理** | Project CRUD，支持 Git / CI / 部署元信息配置，多项目隔离 |
| **Task CRUD** | 任务的创建、列表、详情、删除，关联 Project |
| **9 态状态机** | \`draft → ticket_ready → dispatched → result_submitted → reviewing → changes_requested / approved / rejected → archived\`，含代码级状态跃迁白名单校验 |
| **TaskEvent 审计日志** | 关键操作自动写入事件时间线（创建、状态变更、产物上传、Review 提交） |
| **TaskArtifact 产物管理** | 支持 \`execution_log\` / \`diff\` / \`review_note\` / \`ci_log\` / \`screenshot\` 类型，独立的 storage 模型预留外部文件存储 |
| **ReviewRecord 审查记录** | Review 创建/查询，支持 \`approved\` / \`rejected\` / \`changes_requested\` 决策及 JSON 问题列表 |
| **Submit Result 摘要** | submit-result 时支持附带 \`result_summary\` 文本字段 |
| **PR / CI / Deploy Stub** | 3 个外部集成点保持 Stub 状态，返回 “not implemented in MVP” |

### 前端

| 模块 | 能力 |
|------|------|
| **6 个页面** | Dashboard、ProjectList、ProjectConfig、TaskList、TaskCreate、TaskDetail |
| **6 个组件** | StatusBadge、DiffViewer、ReviewPanel、TicketPreview、EventTimeline、ArtifactTab |
| **Artifact 上传和 diff 展示** | 支持上传产物并在前端查看 |
| **Review reviewer 显示** | 审查者信息在 task detail 中展示 |
