## 1. v0.4 完成范围

### S1: Code Context API

AgentRun 可读取任务代码上下文（可选，由前端传入或 AI Provider 构建）。

- `POST /api/tasks/{id}/code-context` — 上传代码上下文
- `GET /api/tasks/{id}/code-context` — 读取代码上下文
- 上下文按文件组织：path / content / language
- 上下文内容自动 secret redaction
- `UnifiedSandboxProvider` 构建 prompt 时注入 code context

### S2: Patch Apply Sandbox

在沙箱中验证 patch 合法性并预览变更。

- `POST /api/tasks/{id}/agent-runs/{run_id}/sandbox/apply-patch` — 应用 patch
- `GET /api/tasks/{id}/sandbox/patch-results` — 列出沙箱结果
- 沙箱操作：计算 SHA256、增减行数、变更文件统计、检查禁止路径
- 不写数据库 / 不执行文件 IO / 不调用外部 API
- 前端 PatchResultDisplay 组件展示变更摘要

### S3: Sandbox Approval Gate

在沙箱 patch 应用之前的安检关卡。

- `GET /api/tasks/{id}/sandbox/gate` — 只读评估
- `POST /api/tasks/{id}/sandbox/evaluate-gate` — 评估 + 写 TaskEvent
- 检查项：archived_task、no_sandbox_result、not_applied、no_changed_files、forbidden_path、secret_detected、agent_run_unknown、agent_run_not_in_task、risk_too_high、human_required
- 通过条件：所有检查不阻塞 && 无 secret && 无禁止路径 && 风险不超限
- 不写数据库 / 不执行文件 IO / 不调用外部 API

### S4: Review Packet Preview API

Stateless PR review 预览，MockGitHub / MockSonar connector。

- `POST /api/review-packets/preview` — 生成 review packet
- 返回 `packet`（PR/Sonar/pytest/编译/安全边界等 30+ 字段）+ `decision`（review_status / blockers / warnings）
- 22 条检查规则（head 校验、Sonar QG、duplication、hotspots、pytest、compileall、npm build、Playwright、安全边界、代码异味等）
- 统计意义上每返回一次 packet 即完成一次 review（无需显式 submit）
- Mock 数据覆盖 9 种 PR 场景（approved / blocked / needs_update / 404）
- 不读 secret_ref / 不执行 shell / 不访问 Project.root_path / 不调用真实 API
