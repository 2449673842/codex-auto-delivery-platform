## 1. v0.2 完成范围

v0.2 在 v0.1 MVP 基线基础上，增加了 Agent 自动化引擎的后端模型、审批策略、风险评估，以及前端 Agent 管理面板和 Playwright MCP 验收。

## 2. 后端 S1-S4：Agent 后端核心

| 模块 | 说明 |
|------|------|
| **AgentProfile** | Agent 注册、配置、启用/禁用（5 个 API 端点） |
| **AgentRun** | Agent 执行记录创建、状态流转、结果回填（5 个 API 端点） |
| **AgentReview** | AI 审查结论记录（2 个 API 端点） |
| **ApprovalPolicy** | 审批策略 CRUD（5 个 API 端点） |
| **ApprovalDecision** | 审批决策 ORM 模型（等待 S5 评估逻辑接入） |

### AgentRun 状态机

```
queued → running → succeeded / failed / canceled / human_required
```

PATCH 仅允许 queued→running 或 cancel，terminal 状态必须通过 submit-result 端点。

### TaskStatus 新增

新增第 10 个状态 \`human_required\`，位于 reviewing 之后，人工审批后流转到 approved / rejected / changes_requested。

## 3. 后端 S5-S6：审批核心

| 功能 | 说明 |
|------|------|
| **RiskAssessment** | 启发式风险评估引擎（关键词驱动） |
| **evaluate-approval** | POST /api/tasks/{id}/evaluate-approval |
| **auto-approve** | POST /api/tasks/{id}/auto-approve |
| **approval-decisions** | GET /api/tasks/{id}/approval-decisions |
| **effective_policy** | 默认策略始终生效（require_tests_passed, require_no_security_issues） |

### 风险等级规则

| 条件 | 风险等级 |
|------|----------|
| tests_passed=False | high |
| sonar_passed=False | high |
| security_issues_found=True | high |
| token/secret/api key 明文 | critical |
| subprocess/os.system/root_path | critical |
| auth/deploy/database 变更 | high |
| production/trade/payment | critical |

### auto-approve 限制

- 仅 reviewing 状态可执行
- human_required 状态返回 409
- high / critical risk 返回 409
- archive 任务返回 409
