## 1. v0.3 完成范围

### S1: Sandbox Provider

模拟 AI Provider 实现，可在无外部 AI 依赖下完成全链路验证。

- `SandboxProvider` 根据 `run_type` 生成 plan / execute / review / test 输出
- 输出包含 `plan_md`、`patch_diff`、`review_md`、`raw_result_json`
- 不读 `secret_ref`、不执行 shell、不访问 `Project.root_path`

### S2: OpenAI Minimal Adapter

真实 OpenAI API 调用适配器，通过 `OPENAI_API_KEY` 环境变量提供凭据。

- `OpenAIProvider` 调用 GPT-4o-mini chat completions
- 按 `run_type` 构建 prompt 并解析响应到 `AgentRunResult`
- API key 仅从环境变量读取，永不从 `AgentProfile.secret_ref` 读取
- 无 API key 时快速失败，不暴露凭据

### S3: AI Output Governance

AI 输出校验、安全治理、审批决策标记。

- **S3-1**: `validate_agent_run_result()` — 空输出、JSON 合法性、patch diff 结构校验
- **S3-2**: `validate_patch_diff()` — fnmatch 禁止路径匹配（.env, migrations, .pem, .key, credentials, CI config）
- **S3-3**: `parse_review_result()` + `create_agent_review_from_ai_output()` — review.md → AgentReview
- **S3-5**: `check_risk_report()` — risk_level 校验，high/critical → requires_human
- **S3-6**: `build_trace_json()` — 标准化 raw_result_json 结构
- **Secret redaction**: 7 种 pattern（sk-, ghp_/gho_/ghu_/ghs_, AKIA, private key, password, token, api_key）
- **dispatch 集成**: governance 校验已接入 `dispatch_agent_run`；requires_human 自动创建 `ApprovalDecision`

### S4: TaskDetail AI Output Display

前端展示 AI Provider / AgentRun / Governance / Approval 结果。

- AgentRun 治理卡：valid/invalid、风险等级、requires_human、errors/warnings、provider 来源
- 治理状态标签：AI-generated、Sandbox/OpenAI 输出、Secret redacted、Not merged、Not executed
- 审批决策区：human_required、auto_approve_allowed、decision_reason、risk_level
- AI 产物标识
- 不新增 API key 配置、GitHub PR、CI/Sonar/Deploy
