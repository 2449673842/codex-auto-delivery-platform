## 1. 背景

v0.2 完成了 Agent 记录式框架（AgentProfile / AgentRun / AgentReview / ApprovalDecision）和自动循环编排（orchestration），但 AgentRun 仍然是纯记录式的——创建 run、标记状态、回填结果均由外部手动完成。

v0.3 的目标是：将 AgentRun 从记录式升级为**受控执行框架**。第一阶段只实现 **Sandbox Provider**（模拟 AI 输出），不调用真实外部 AI API。

## 2. 架构

### 2.1 Provider 抽象层

```
┌──────────────────────────────────────────────┐
│            orchestration_service              │
│  (decide next action → dispatch AgentRun)     │
└──────────────────┬───────────────────────────┘
                   │
                   ▼
┌──────────────────────────────────────────────┐
│           ai_provider_service.py              │
│  dispatch_agent_run(run_id) → AgentRunResult  │
└──────────────────┬───────────────────────────┘
                   │
        ┌──────────┴──────────┐
        ▼                     ▼
┌───────────────┐   ┌───────────────────┐
│ Sandbox       │   │ Future: Codex     │
│ Provider      │   │ Future: Claude    │
│ (v0.3)        │   │ Future: OpenAI    │
└───────────────┘   └───────────────────┘
```

### 2.2 Provider 接口

所有 Provider 实现统一的抽象接口：

```
class AiProvider(ABC):
    @abstractmethod
    async def execute(self, run: AgentRun) -> AgentRunResult: ...
```

`AgentRunResult` 包含：

```python
class AgentRunResult(BaseModel):
    output_summary: str
    output_log: str
    raw_result_json: str
    plan_md: str | None = None
    patch_diff: str | None = None
    review_md: str | None = None
    risk_report: dict[str, Any] | None = None
```

### 2.3 执行流程

```
orchestration step → dispatched + no AgentRun
  → create AgentRun (queued)
  → AiProvider.dispatch_agent_run(run.id)
    → agent_run_service.update_status(queued → running)
    → provider.execute(run)
    → agent_run_service.update_status(running → succeeded)
    → agent_run_service.submit_result(output_summary, output_log, raw_result_json)
    → task_service.submit_result (dispatched → result_submitted)
    → 继续编排流程
```

## 4. Sandbox Provider 实现

### 4.1 输入

从 AgentRun 和 Task 中读取：

- `Task.title` / `Task.description` → 用于构造模拟结果
- `AgentRun.run_type` → plan / execute / review / test / remediate
- `AgentRun.input_prompt` → 执行提示

### 4.2 输出（run_type 对应的产物）

| run_type | output_summary | output_log | raw_result_json |
|----------|---------------|------------|-----------------|
| plan | 计划摘要 | 执行步骤枚举 | `{"plan_md": "..."}` |
| execute | 执行结果摘要 | 执行日志 | `{"patch_diff": "...", "files_changed": []}` |
| review | 审查结论 | 审查过程 | `{"decision": "approved", "issues": []}` |
| test | 测试结果 | 测试日志 | `{"tests_passed": true, "coverage": 85}` |
| remediate | 修复摘要 | 修复步骤 | `{"fixes": []}` |

### 4.3 模拟生成规则

- 不调用外部网络
- 不读取真实 API key
- 不使用 LLM / AI 库
- 基于 Template 字符串 + run_type 组合生成合理文本
- 产物包含 `plan.md`、`patch.diff`、`review.md` 等受控文件内容

## 5. 接入现有流程

### 5.1 orchestration 集成

在 `orchestration_service.py` 的 `_do_step_create_agent_run` 中，创建 AgentRun 后立即调用 Provider：

```python
# 创建 AgentRun 后
await ai_provider_service.dispatch_agent_run(db, run.id, actor)
# dispatch 内部：queued → running → succeeded + submit_result
# 返回后 task 已在 result_submitted 状态
```

### 5.2 条件

- 仅当 sandbox provider 启用时自动执行
- 可配置禁用（用于测试或人工模式）

## 6. 测试要求

### 6.1 Sandbox Provider

1. Sandbox Provider 正常生成 plan 产物
2. Sandbox Provider 正常生成 execute 产物（含 patch.diff）
3. Sandbox Provider 正常生成 review 产物
4. Sandbox Provider 不读取 secret_ref
5. Sandbox Provider 不访问 Project.root_path
6. Sandbox Provider 不执行 shell / subprocess

### 6.2 编排集成

7. dispatch_agent_run 将 AgentRun 从 queued 推进到 succeeded
8. dispatch_agent_run 写入 output_summary / output_log / raw_result_json
9. dispatch_agent_run 创建 TaskArtifact
10. succeeded 后 step 继续推进到 result_submitted
11. 完整编排：dispatched → AgentRun → succeeded → result_submitted → reviewing

### 6.3 安全边界

12. Sandbox Provider 不调用外部 AI API
13. Sandbox Provider 不 git commit / push
14. Sandbox Provider 不创建真实 GitHub PR
15. Sandbox Provider 不调用 CI
16. Sandbox Provider 不调用 Sonar API
17. high/critical risk 不允许 auto approve
18. human_required 不允许 auto approve

## 7. 禁止事项

| 禁止 | 原因 |
|------|------|
| 调用真实 Codex / Claude / OpenAI API | 安全边界，留待后续版本 |
| 读取 secret_ref 真实环境变量值 | 安全边界 |
| 写项目目录（Project.root_path） | 安全边界 |
| 执行 shell / subprocess / os.system | 安全边界 |
| git clone / commit / push | 安全边界 |
| 创建真实 GitHub PR | 方向 B，暂缓 |
| 调用真实 CI | 方向 C，暂缓 |
| 调用真实 Sonar API | 方向 C，暂缓 |
| 部署 | 安全边界 |

## 8. 开放问题

1. Sandbox Provider 是否默认启用？建议默认启用。
2. Provider 选择由 AgentRun 的 provider 字段还是新配置决定？建议复用 `AgentProfile.provider` 字段，新增 `sandbox` 类型。
3. dispatch_agent_run 是同步 await 还是后台任务？建议同步 await（Sandbox 很快，无需异步）。
4. orchestration step 中 Sandbox Provider 失败时怎么处理？建议 AgentRun → failed，停止编排。
