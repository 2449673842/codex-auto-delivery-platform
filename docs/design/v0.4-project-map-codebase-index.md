# v0.4 S7 — Project Map & Codebase Index

> Design document for the lightweight Project Map & Codebase Index layer.
> First version: human-maintainable, audit-friendly, no vector DB, no automated scanner.

---

## 1. Problem Statement

AI (Browser AI, Multi-AI Collaboration, AI provider calls) currently wastes time and tokens:

- Repeatedly searches for which file is which module
- Re-discovers module responsibilities every task
- Has no centralized mapping from "feature X" to "file Y"
- Cannot quickly answer "what router/service/schema/test files belong to this feature?"
- Context Selector has no structured index to decide what files to include

Without a Project Map, every AI session starts with costly file traversal.

---

## 2. Solution

Add three lightweight, human-and-AI-readable artifacts under `docs/project-map/`:

| Artifact | Format | Audience | Purpose |
|----------|--------|----------|---------|
| `codebase-index.md` | Markdown | Human + AI | Full project walkthrough: module descriptions, file roles, task-to-file mapping |
| `repository-map.json` | JSON | Programmatic | Machine-readable index for Context Selector, CI checks, future tooling |
| `update-policy.md` | Markdown | Human + AI | Rules for when and how to update the Project Map |

---

## 3. Design Principles

- **Readable by both humans and AI** — no binary/proprietary formats
- **Manually maintainable** — first version written by hand, not auto-scanned
- **No new dependencies** — no vector DB, no indexer, no scanner
- **Auditable** — every change tracked via git diff
- **Stable prefix** — top-level structure changes rarely
- **Minimum viable** — enough to eliminate file discovery wasted effort, not a full knowledge base
- **Not a replacement** for Code Context (which sends actual file content)

---

## 4. Relation to Existing Components

```text
Project Map:       tells AI which files to look at
Code Context:      sends actual file content to AI
Context Selector:  uses Project Map to pick minimal file set
Review Packet:     can verify Project Map freshness against changed files
```

### Flow (future)

```
Task created
→ Context Selector reads Project Map (repository-map.json)
→ Identifies relevant modules based on task description
→ Builds Code Context from minimal file set
→ AI Provider / Browser AI consumes minimal context
→ Post-merge: update Project Map if changed files trigger update policy
```

## 6. update-policy.md Triggers

The Project Map **must** be updated when a PR changes:

- Any file under `backend/app/routers/` — new or modified router
- Any file under `backend/app/services/` — new or modified service
- Any file under `backend/app/schemas/` — new or modified schema
- Any file under `backend/app/models/` — new or modified model
- Any new test file under `backend/tests/`
- Any new API endpoint
- Any new artifact type in `enums.py`
- Any new event type in `enums.py`
- Any new task status in `enums.py`
- Any new provider in `enums.py`
- Any new page/component under `frontend/src/pages/` or `frontend/src/components/`
- Any new frontend service or store
- Any update to `AGENTS.md` safety boundaries
- Any update to `docs/design/` that introduces new core concepts

**Not required** for:
- Pure bugfixes with no new file
- Typo fixes
- Comment changes
- Test-only changes to existing tests (no new test file)
- Frontend style-only changes

---

## 7. PR Body Integration

Add to PR body requirements (`AGENTS.md`):

| Field | Required | Notes |
|-------|----------|-------|
| Project Map updated? | ✅ | yes/no/not_needed |

If changed files include new router/service/schema/model/test files but Project Map says "no", mastermind can mark `needs_update`.

---

## 8. Non-goals (v0.4 S7)

- No automated file scanner / indexer
- No vector database
- No embedding / semantic search
- No real GitHub API calls
- No CI/Sonar/Deploy integration
- No backend code changes
- No frontend code changes
- No database migration
- No shell/subprocess execution
- No `Project.root_path` access
- No secret reading
- No real file system scanning

---

## 9. Future Extensions (not now)

- Automated staleness detection in Review Packet
- Context Selector integration using repository-map.json
- CI check: "did you forget to update Project Map?"
- Browser AI consuming Project Map as part of its context packet
