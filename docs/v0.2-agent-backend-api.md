## 4. AgentRun 状态机说明

```
ALLOWED_AGENT_RUN_TRANSITIONS = {
    "queued": ["running", "canceled"],
    "running": ["succeeded", "failed", "canceled", "human_required"],
    "succeeded": [],       # 终态
    "failed": [],          # 终态
    "canceled": [],        # 终态
    "human_required": [],  # 终态
}
```

### PATCH 限制

- PATCH /agent-runs 仅允许：queued -> running，或 queued/running -> canceled
- terminal 状态（succeeded / failed / human_required）必须通过 submit-result 端点
- 非法 PATCH 返回 409

### submit-result 端点

- 接受 status=succeeded 或 status=failed
- 自动创建 TaskArtifact（diff -> artifact_type=diff，log -> artifact_type=execution_log）
- 写入 agent_run_succeeded 或 agent_run_failed 事件
- 提交后状态不可再变更（终态）

## 5. TaskStatus human_required 状态说明

v0.2 在原有 9 态基础上新增第 10 个状态 `human_required`。

```
ALLOWED_TRANSITIONS = {
    ...
    "reviewing": ["approved", "rejected", "changes_requested", "human_required"],
    "human_required": ["approved", "rejected", "changes_requested"],
    "approved": ["archived"],
    "rejected": ["archived"],
    ...
}
```

### 规则

- human_required 不能直接 archived（返回 409）
- 必须先由人工 approve / reject / request-changes 后才能归档
- human_required 写入两个 TaskEvent：status_changed + human_approval_required

## 6. require-human-approval 使用说明

**端点**：`POST /api/tasks/{id}/require-human-approval`

**请求体**：`{ "actor": "...", "message": "..." }`

**前置状态**：reviewing

**效果**：

| 效果 | 说明 |
|------|------|
| Task 状态 | reviewing -> human_required |
| TaskEvent | status_changed（状态机跃迁） |
| TaskEvent | human_approval_required（额外审计） |

**后续操作**：

- `POST /api/tasks/{id}/approve` -> human_required -> approved
- `POST /api/tasks/{id}/reject` -> human_required -> rejected
- `POST /api/tasks/{id}/request-changes` -> human_required -> changes_requested

**非法操作返回 409**：
- human_required -> archive
- draft -> require-human-approval

## 7. AgentRun submit-result 使用说明

**端点**：`POST /api/tasks/{id}/agent-runs/{run_id}/submit-result`

**请求体**：

```json
{
  "status": "succeeded",
  "output_summary": "全部测试通过",
  "output_diff": "--- a/file.py\n+new line",
  "output_log": "Build OK",
  "duration_ms": 300000,
  "error_message": null
}
```

**约束**：

- 仅 running 状态可 submit-result
- 状态变为 succeeded/failed 后不可再变更（终态）
- diff 摘要存 AgentRun.output_diff，完整 diff 自动创建 TaskArtifact（type=diff）
- log 摘要存 AgentRun.output_log，完整 log 自动创建 TaskArtifact（type=execution_log）
- 自动写入 agent_run_succeeded 或 agent_run_failed 事件
- run_id 不属于 task_id 时返回 404

## 8. AgentReview 使用说明

**端点**：`POST /api/tasks/{id}/agent-runs/{run_id}/review`

**请求体**：

```json
{
  "reviewer_agent_id": 1,
  "decision": "approved",
  "risk_level": "low",
  "comments": "代码质量良好",
  "issues_json": "[{\"severity\":\"major\",\"file\":\"src/main.py\",\"line\":42}]",
  "confidence_score": 0.85
}
```

**约束**：

- AgentReview 是 AI 预审批建议，不等同于人类最终审批（ReviewRecord）
- run_id 必须属于 task_id，否则返回 404
- confidence_score 范围 0.0 ~ 1.0，越界返回 422
- risk_level 仅接受 low / medium / high / critical，非法值返回 422
- 写入 agent_review_submitted 事件

## 10. 安全边界

| 规则 | 说明 |
|------|------|
| secret_ref 不存真实 key | 只存环境变量名，v0.2 不读取 |
| 不调用外部 AI | 所有 Agent 操作为记录式 |
| 不执行 shell | 无 subprocess/os.system |
| 不访问 Project.root_path | 无文件系统操作 |
| archived Task 保护 | 禁止创建 AgentRun（409） |
| AgentRun 归属校验 | run_id 不属于 task_id 时返回 404 |
| AgentReview 归属校验 | run_id/task_id 不匹配返回 404 |
| Pydantic 输入校验 | agent_type/provider/status/risk_level 均受 pattern 约束 |
| 非法状态跃迁 | AgentRun 和 TaskStatus 均返回 409 |
