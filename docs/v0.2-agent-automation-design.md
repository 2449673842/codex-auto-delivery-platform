## 1. v0.2 目标

平台从「人工复制任务单给外部 AI」升级为「AI Agent 自动编排平台」，核心新增：

| 模块 | 说明 |
|------|------|
| **AgentProfile** | 注册和管理 AI Agent（Planner / Executor / Reviewer / Test），不保存明文 API key |
| **AgentRun** | Agent 执行记录——每次执行的输入、输出、耗时、状态 |
| **AgentReview** | AI Reviewer 的审查结论记录 |
| **ApprovalPolicy** | 自动审批策略——什么条件下 AI 可自主通过 |
| **RiskAssessment** | 风险等级评估——决定是否允许自动审批 |
| **自动执行记录** | AgentRun 记录 diff / log / summary，大内容走 TaskArtifact |
| **自动审批建议** | 基于 RiskAssessment + ApprovalPolicy 输出 auto-approve / human-required |
| **人工审批兜底** | high / critical 风险必须人工审批，main merge 必须人工审批 |

### 核心原则

v0.2 先做**编排与记录**，不直接无监管修改生产代码。所有 Agent 动作写入审计日志，所有自动审批可追溯、可拦截。

## 2. Agent 角色模型

### 2.1 四类 Agent

| 角色 | 标识符 | 职责 |
|------|--------|------|
| **Planner Agent** | `planner` | 拆解需求、生成结构化任务单、规划执行步骤 |
| **Executor Agent** | `executor` | 读取任务单、执行代码修改、产生 diff + log |
| **Reviewer Agent** | `reviewer` | 代码审查、安全审查、质量审查、输出审查结论 |
| **Test Agent** | `test` | 执行或记录测试结果（Playwright / unit / integration） |

### 2.2 AgentProfile 表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTO | 自增主键 |
| `name` | TEXT NOT NULL UNIQUE | Agent 名称（如 `codex-planner-v1`） |
| `agent_type` | TEXT NOT NULL | `planner` / `executor` / `reviewer` / `test` |
| `provider` | TEXT NOT NULL | `codex` / `openai` / `claude` / `local` / `manual` |
| `model_name` | TEXT | 模型标识（如 `deepseek-v4-pro`、`gpt-4o`） |
| `enabled` | BOOLEAN DEFAULT `1` | 是否启用 |
| `max_runtime_seconds` | INTEGER DEFAULT `300` | 单次最大运行时间 |
| `max_attempts` | INTEGER DEFAULT `3` | 最大重试次数 |
| `allowed_projects` | TEXT | 允许执行的项目 ID 列表（JSON 数组，空=全部） |
| `created_at` | DATETIME DEFAULT now | 创建时间 |
| `updated_at` | DATETIME DEFAULT now | 最后更新时间 |

> **安全规则**：不保存 API key。只保存环境变量名或 secret 引用名（如 `CODEX_OPENAI_KEY`）。实际 key 由运行时的环境变量提供。

| 字段 | 类型 | 说明 |

| `branch` | TEXT | 执行时分支 |
| `commit_sha` | TEXT | 执行结果 commit hash |
| `pr_url` | TEXT | 关联 PR 链接（预留） |
| `risk_level` | TEXT DEFAULT `low` | `low` / `medium` / `high` / `critical` |
| `attempt_no` | INTEGER DEFAULT `1` | 当前尝试次数（第几次重试） |

| `finished_at` | DATETIME | 结束时间 |

| `error_message` | TEXT | 错误信息 |
| `raw_result_json` | TEXT | Agent 原始返回 JSON |
| `created_at` | DATETIME DEFAULT now | 创建时间 |
| `updated_at` | DATETIME DEFAULT now | 最后更新时间 |

### 存储策略

- `output_diff` / `output_log`：AgentRun 只保存**摘要**（前 2000 字符）
- 完整 diff / log 通过 TaskArtifact 存储，artifact_type 复用：`diff` / `execution_log`
- `raw_result_json`：Agent 原始返回，按需解析

## 4. AgentReview 数据模型

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTO | 自增主键 |
| `task_id` | INTEGER FK → Task.id | 关联任务 |
| `agent_run_id` | INTEGER FK → AgentRun.id | 关联的 Agent 执行 |
| `reviewer_agent_id` | INTEGER FK → AgentProfile.id | Reviewer Agent |
| `decision` | TEXT NOT NULL | `approved` / `changes_requested` / `rejected` / `human_required` |
| `risk_level` | TEXT DEFAULT `low` | 本次审查评估的风险等级 |
| `comments` | TEXT | 审查意见 |
| `issues_json` | TEXT | 发现的问题列表（JSON 数组） |
| `confidence_score` | REAL | 置信度 0.0~1.0 |
| `created_at` | DATETIME DEFAULT now | 创建时间 |

### 与 ReviewRecord 的关系

| 维度 | ReviewRecord（v0.1） | AgentReview（v0.2） |
|------|---------------------|---------------------|
| 审查者 | 人工 | AI Agent |
| 决策 | approved/rejected/changes_requested | 同上 + human_required |
| 风险等级 | 无 | 有 risk_level |
| 置信度 | 无 | 有 confidence_score |
| 用途 | 最终人类审批 | AI 预审批建议 |

> **AI Reviewer 的结论不能等同于最终人类审批**。AgentReview 作为审批建议流入 ApprovalPolicy 判断。

## 5. ApprovalPolicy 自动审批策略

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | INTEGER PK AUTO | 自增主键 |
| `name` | TEXT NOT NULL | 策略名称（如 "默认安全策略"） |
| `project_id` | INTEGER FK → Project.id | 关联项目（NULL=全局） |
| `enabled` | BOOLEAN DEFAULT `1` | 是否启用 |
| `max_risk_level_for_auto_approve` | TEXT DEFAULT `low` | 允许自动审批的最高风险等级 |
| `require_tests_passed` | BOOLEAN DEFAULT `1` | 是否要求测试全部通过 |
| `require_sonar_passed` | BOOLEAN DEFAULT `0` | 是否要求 Sonar 通过（预留） |
| `require_no_security_issues` | BOOLEAN DEFAULT `1` | 是否要求无安全问题 |
| `allow_auto_approve_docs_only` | BOOLEAN DEFAULT `1` | 纯文档修改是否允许自动审批 |
| `allow_auto_approve_frontend_style_only` | BOOLEAN DEFAULT `1` | 纯前端样式修改是否允许自动审批 |
| `forbid_auto_merge_main` | BOOLEAN DEFAULT `1` | 禁止自动 merge main |
| `forbid_auto_deploy_prod` | BOOLEAN DEFAULT `1` | 禁止自动部署生产 |
| `created_at` | DATETIME DEFAULT now | 创建时间 |
| `updated_at` | DATETIME DEFAULT now | 最后更新时间 |

### 审批决策矩阵

| 条件 | 结果 |
|------|------|
| risk=low + 测试通过 + 无安全问题 + 非 main merge | **自动 approved** |
| risk=low + 纯文档/样式修改 + 非 main merge | **自动 approved** |
| risk=medium | **AI 预审批 + 人工确认** |
| risk=high / critical | **必须人工审批** |
| 涉及 merge main | **必须人工审批** |
| 涉及生产部署 | **必须人工审批** |

## 6. 风险评估规则

RiskAssessment 模块分析 AgentRun 的输出，自动计算风险等级。

### 评估因子

| 因子 | 影响 | 检测方式 |
|------|------|----------|
| 修改数据库模型 | +high | diff 含 `models/`、`ALTER TABLE`、`migration` |
| 修改认证/权限 | +high | diff 含 `auth`、`login`、`permission`、`token` |
| 修改部署配置 | +high | diff 含 `docker-compose`、`Dockerfile`、`deploy`、`k8s` |
| 删除文件 | +medium | diff 含 `deleted file` |
| 大规模重构 | +medium | diff > 500 行或修改 > 10 文件 |
| 修改 CI/CD | +medium | diff 含 `.github/workflows`、`Jenkinsfile` |
| 修改交易/资金/生产逻辑 | +critical | diff 含 `trade`、`order`、`payment`、`money`、`fund` |
| 涉及 root_path 文件操作 | +high | diff 含 `os.path`、`shutil`、`open()` 写模式 |
| 引入新依赖 | +low | diff 含 `requirements.txt`、`package.json` 新增 |
| 测试失败 | +medium | TestRun status=failed |
| Sonar 失败 | +medium | 预留 |

### 输出

```
{
  "risk_level": "medium",
  "risk_reasons": ["修改数据库模型", "大规模重构"],
  "human_required": true
}
```

## 7. 自动循环流程

### 完整流程

```
Task (ticket_ready)
  │
  ├── 1. 创建 Executor AgentRun (queued)
  │      └── Agent 执行 → status = succeeded/failed
  │
  ├── 2. Agent 回填 output_diff / output_log / output_summary
  │      └── 完整 diff → TaskArtifact
  │
  ├── 3. 触发或记录测试（可选）
  │      └── TestRun 创建
  │
  ├── 4. 测试失败 → changes_requested → 自动创建 remediation AgentRun
  │      └── 重试，最多 max_attempts 次
  │      └── 超过上限 → human_required
  │
  ├── 5. 测试通过 → Reviewer AgentRun
  │      └── AI Review → AgentReview 记录
  │
  ├── 6. RiskAssessment 评估风险等级
  │
  ├── 7. ApprovalPolicy 判断
  │      ├── 可自动审批 → Task 状态 = approved
  │      └── 不可自动审批 → Task 状态 = human_required
  │
  └── 8. 人工 Approve → archived
```

### 循环限制

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_attempts` | 3 | 最大自动重试次数 |
| `max_loop_minutes` | 60 | 单任务最大自动循环时长 |
| `cooldown_seconds` | 30 | 两次 AgentRun 间隔 |

> **禁止无限循环**。达到上限后任务状态 → `human_required`，等待人工介入。

### 8.4 注意

v0.2 先不真正调用外部 AI API。所有 AgentRun 创建、结果回填和审批策略判断均在平台内完成，Agent 输出由人工或外部工具粘贴。

## 11. 安全边界

| 规则 | 说明 |
|------|------|
| 不保存明文 API key | AgentProfile 只存环境变量名引用，实际 key 由运行时环境提供 |
| 不允许 AI 自动 merge main | `forbid_auto_merge_main` 默认开启 |
| 不允许 AI 自动部署生产 | `forbid_auto_deploy_prod` 默认开启 |
| 不允许 AI 删除项目目录 | 所有 AgentRun 不做文件系统操作 |
| 不允许 AI 访问 root_path | Project.root_path 仅展示，Agent 不读不写 |
| 不允许 AI 执行危险 shell | Agent 输出只做文本记录，不自动执行 |
| 不允许无限循环 | `max_attempts=3` 硬编码上限 |
| 高风险任务必须人工审批 | risk=high/critical → human_required |
| 所有 AgentRun 输入输出审计 | input_prompt / output_summary / diff / log 全记录 |
| 所有自动审批可追溯 | 每个 auto-approve 写 TaskEvent + ApprovalPolicy 快照 |
| 自动审批只适用于低风险任务 | `max_risk_level_for_auto_approve=low` 默认 |
| archived 保护 | archived Task 拒绝所有 AgentRun 创建 |

## 设计决策说明

### 为什么 AgentReview 独立于 ReviewRecord？

ReviewRecord（v0.1）记录的是**人类最终审批结论**。AgentReview（v0.2）记录的是**AI 预审批建议**，含置信度和风险等级，用于 ApprovalPolicy 的判断输入。语义不同，分开更清晰。

### 为什么 AgentRun 不直接调用 AI API？

v0.2 是编排层设计，先确保所有数据模型、审批策略、审计追踪就绪。真实调用 AI API（v0.3）需要 token 管理、成本控制、超时重试、并发限制等基础设施，不应与数据模型耦合。

### 为什么 ApprovalPolicy 按风险等级而不是按文件路径？

文件路径规则过于脆弱——同路径的修改可能有完全不同的风险。风险等级基于修改内容的关键词分析 + 测试结果 + 文件类型，更贴近实际风险。

### 为什么 max_attempts 硬编码而不是可配置？

硬编码 3 次是安全底线。可配置的循环次数可能被误设为无限，导致 Agent 无限重试。后续版本可在 ApprovalPolicy 中增加 `max_attempts` 字段，但平台需硬性限制上限。
